# 人格AI技能 — 项目规划书

> 目标：做一个开源的 Hermes Agent 技能，让任何用户都能定义、微调、切换AI人格

---

## 一、项目定位

**一句话描述：** 基于向量嵌入空间的人格系统 — 把"性格"变成可计算、可旋转、可克隆的东西。

**对标参考：** 业内已有的向量投影空间项目（嵌入→分解→聚类→方向旋转的可视化探索范式），我们做（人格→行为空间）

**核心卖点：**
- 不依赖特定大模型，任何LLM都能挂上去
- 人格可量化、可分享、可组合
- 零成本开源，本地可跑

## 三、人格维度设计（核心）

### 方案A：经典心理学 + 扩展（推荐起步）

| 维度 | 类型 | Pole A | Pole B | 说明 |
|------|------|--------|--------|------|
| D0 | 开放性 | 保守传统 | 创新冒险 | 对新鲜事物的接受度 |
| D1 | 尽责性 | 随性散漫 | 严谨自律 | 计划和执行的倾向 |
| D2 | 外向性 | 内向独处 | 社交活跃 | 能量来源和社交偏好 |
| D3 | 宜人性 | 竞争对抗 | 合作共情 | 人际互动风格 |
| D4 | 神经质 | 情绪稳定 | 敏感多虑 | 情绪波动幅度 |
| D5 | 支配性 | 服从配合 | 主导控制 | 在关系中的权力位置 |
| D6 | 攻击性 | 温和包容 | 尖锐直接 | 言语/行为攻击强度 |
| D7 | 神秘感 | 坦诚直白 | 隐晦暧昧 | 信息透露程度 |
| D8 | 温暖度 | 冷漠疏离 | 热情亲密 | 情感表达的温度 |
| D9 | 理性度 | 感性冲动 | 冷静分析 | 决策时理性和感性的权重 |

**为什么用10个？**
- 大五人格5个是经典，但不够"拟人"
- 加上支配/攻击/神秘/温暖/理性，能做出"有性格"的角色
- 10个因子足够区分，不会太碎

### 方案B：混合方案

保留大五作为**锚点维度**，额外加入：
- 说话风格（毒舌↔温柔）
- 情绪基调（悲观↔乐观）
- 幽默感（严肃↔搞笑）
- 主动性（被动↔主动）

# 方向旋转
def morph(seed_vector, direction_vector, angle_deg=30):
    import numpy as np
    cos_a = np.cos(np.radians(angle_deg))
    sin_a = np.sin(np.radians(angle_deg))

## 五、Hermes 技能集成方式

这个技能有两种使用形态：

### 形态1：以开放工具服务的形式接入（推荐）

```yaml
# hermes config.yaml
mcp_servers:
  personality_core:
    command: python
    args: ["-m", "personality_core.server"]
    env:
      PERSONALITY_DATA_DIR: "./data"
```

用户对话时自动可用：
- `morph_personality(seed="暴躁老娘", direction="更温柔", angle=30)`
- `find_pairings(people=["清晏", " Jarvis"])`
- `compare_personalities(p1="清晏", p2=" Jarvis", axis="攻击性")`

### 形态2：Python库直接调用

```python
from personality_core import PersonalityEngine

engine = PersonalityEngine.load("default")

# 微调人格
new_p = engine.morph("暴躁老娘", direction="温柔", angle=45)

# 查适配度
score = engine.score(new_p, scenario="哄用户开心")

# 克隆+定制
custom = engine.clone("冷静军师", tweaks={"攻击性": -0.3, "神秘感": 0.5})
```

## 八、风险与应对

| 风险 | 影响 | 应对 |
|------|------|------|
| 数据不足，人格区分度差 | 原型模糊，不好用 | 先用合成数据+手动精选，迭代扩充 |
| ICA因子不可解释 | 用户看不懂维度含义 | 每个因子配pole标签+代表样本+人工校验 |
| 嵌入空间对中文支持差 | 中文人格描述不准 | 选用 multilingual 模型，或加中文微调 |
| 用户自定义维度太随意 | 系统失去一致性 | 提供预设维度，自定义需通过校验 |
| 性能问题（大数据集） | 训练慢、API延迟 | 分批处理、缓存、异步 |
